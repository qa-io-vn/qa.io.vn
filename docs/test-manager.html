Test Manager

Policy · People · Process

Mission: own the test organization — policy, strategy, people, budget, process maturity, and the quality story told to executives. You work across releases, not inside one.

ISTQB home turf: CTAL-TM strategic level · Expert Test Management · Improving the Test Process
11 commands in your kit across releases, not inside one the quality story for execs

Foundations — once, then reviewed yearly

Set the organization's direction.

These four are standing documents — the top of the hierarchy every release inherits. Review them yearly, not every sprint.

test manager — foundations
> /qa:test-policy                  # 1 · WHY the org tests — objectives, top of the document hierarchy
> /qa:create-strategy             # 2 · HOW this product is tested — levels, types, techniques, environments
> /qa:automation-strategy         # 3 · the automation program: gTAA, level split, ROI model
> /qa:tool-select "API test tool"   # 4 · tooling decisions: criteria, comparison, pilot plan

When the numbers look wrong

From symptom to funded fix.

test manager — diagnose
> /qa:quality-report        # escaped defects trending up? →
> /qa:process-improvement   # find the process root cause (late static testing? thin risk coverage?)
> /qa:risk-assessment       # re-aim the effort
> /qa:cost-of-quality       # quantify the failure cost to fund the fix
Escaped defects trending up isn't a testing failure to punish — it's a process root cause to find and fund.The management lens
