/qa:usability-test

Usability evaluation

Evaluate UX with ISTQB usability methods, heuristics, and ISO/IEC 25010 sub-characteristics — distinct from accessibility.

USAGE/qa:usability-test <feature / flow>
Test execution — functional & non-functionalCT-UT (Specialist) · CTAL-TA · ISO/IEC 25010 Usability · CTFL v4.0 Principle 7Manual

What it does

Assesses how usable a feature or flow is: scores it against each ISO/IEC 25010 usability sub-characteristic, runs a heuristic evaluation with severity-rated violations, defines usability task scenarios with success criteria, and reviews user-error protection. Advisory only — does not replace real-user testing. Writes a findings document, modifies no code.

When to use it

To assess how usable a feature or flow is (UX, not WCAG accessibility).

Prerequisites

qa.config.yml for paths.docs_dir, risk_areas, project.* context; a target feature/flow.

Output

Non-functional usability evaluation report (29119-3 usability Test Execution/analysis result) -> docs_dir/USABILITY-<feature>.md.

Mechanics

How it works

  1. Read config; resolve the target flow (default risk_areas.critical when empty)
  2. Assess against each ISO/IEC 25010 usability sub-characteristic
  3. Run a heuristic evaluation; rate each violation 0-4 and map to a sub-characteristic
  4. Define usability task scenarios with explicit success criteria
  5. Review forms/flows for user-error protection
  6. Assemble the report and separate heuristic findings from what needs real-user sessions

Why it works

The theory behind it

Usability evaluation (CT-UT; CTAL-TA quality-characteristic analysis) against ISO/IEC 25010 usability. Directly addresses Principle 7 — a defect-free product can still fail its users.

CT-UT (Specialist) · CTAL-TA · ISO/IEC 25010 Usability · CTFL v4.0 Principle 7

Example

See it in use

usability-test
> /qa:usability-test "guest checkout"

✓ Correct when It applies usability heuristics plus task scenarios and stays distinct from accessibility.
